2. localStorage Helpers (Customer Extras)

Customer extra data (email, DOB, address, flags) is stored in localStorage under key "bagicha_customer_extras".

FunctionPurposeStorage key
loadExtras() Reads and parses all customer extras from localStorage. Returns empty object on error. bagicha_customer_extras
saveExtras(data) Serialises and writes updated extras map back to localStorage. bagicha_customer_extras
defaultExtra() Returns a blank CustomerExtra object with all empty strings and notificationEnabled: true. —

6. Event Handlers inside CustomerDashboard

HandlerTriggered byWhat it does
handleSaveExtra(customer, data) EditCustomerModal Save button Merges new CustomerExtra data into the extras map keyed by customer.key. The useEffect then persists to localStorage automatically.
handleToggleNotif(key) Notification Status button in listing row Flips notificationEnabled for the given customer key. If no extras exist yet, starts from defaultExtra().
handleSendWA(template) (inside CustomerDrawer) WhatsApp template buttons in the drawer Calls sendViaWhatsAppWeb(phone, template, name) to open a pre-filled WhatsApp Web link. Sets waSent state for 2.5s to show "Opened!" feedback.

Key constraint: customer.key is the primary identifier — it equals phone if available, else name. All extras, automation logs, and notification prefs are keyed on this value.
